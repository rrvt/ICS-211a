ICS_211a Operations

Starting the ICS_211a

This program has only one purpose, collection of attendence information at events, i.e. ICS-211 information. When starting the program the first thing it does is ask for a file in which to store the data.

If the file chosen is a new file then the program asks for the Event Information. Then it organizes itself to accept barcodes from the reader. The individual reads are displayed in the application window with the most recent one first.

If the BarCode reader has not been plugged in, a reminder message box will be displayed. Otherwise, the program is ready to read barcodes and announces that fact.

If the file exists it is assumed to a file created by this program and the information is read into the internal data structures as if it had never left them. The progam may be exited and restarted whenever it is idle (not actually updating the roster). If the program is restarted with a xxx.211 file, then the contents of that file is displayed in the main window.

Tool Bar, Left to Right

Open Roster

Open a ICS-211 File.

Edit Event Information command

The Event Information appears on the screen after the output is prepared, however it may need to be tweeked. This command starts a dialog box with allows the Event Information to be changed.

Start Barcode Reader and Display Log

This command starts the barcode reader and displays the log. Used after creating reports to restart the barcode reader.

Member Dialog Box

The M(ember) command starts a dialog box which has two sections, Check-In and Check-Out. The Check-In combo (drop-down) box contains all the members information. The Check-In drop-down list contains all the members, the Check-Out list contains only those members that have check-in. Clicking OK adds the members information to the roster (the list of attendees) along with the current date/time.

Visitor Dialog Box

The V(isitor) command starts another dialog box. This one has two sections, Check-In and Check-Out. The Check-In section has edit fields for Call Sign, First and Last Name, and Agency. The Check-Out section has a combo (drop-down) box which list those visitor that have check-in. Clicking OK adds the members information to the roster (the list of attendees) along with the current date/time.

Preparation of the Spreadsheet Command

Once you are finished with the meeting or the next day or two and the spreadsheet output needs to be prepared. The essence is that it will find the first and last record for each attendee. This information is then displayed in the main window in the ICS-211 format. The display is composed of heading information and one line for each attendee. It will become obvious which members did not check out and the Fix Checkout Command will allow a checkout time to be supplied (see details below).

Fix Checkout Command

Only the entries with missing checkouts or an elapsed time less than a quarter hour are displayed here. There is a dropdown box with the callsign and name of the attendee, a Check Out Date and Time edit box, and a Mediian or Average CheckOut time information line. The idea is that the user pick an attendee from the drop down box and a time or time and date.

The Median check out date/time or the average (if there are 2 or fewer attendees) is displayed. This can give the user a hint of what to use for a check out time.

The data structures are update whenever the time edit box or the drop down list is left. It does not matter which order things are done. For example, a single check out time can be chosen and then each attendee can be selected one at a time to be updated with that time. If an attendee is not selected in the dropdown box the spreadsheet entry will not be changed.

Output to a CSV File

The write to the spreadsheet input file is done right after a file is chosen. It will be a comma Separated Value (csv) file including all the elements on display.

Output Current Screen to a Text File

The current screen is written to a text file with a name chosen by appending the nature of the screen to the name of the 211 file. There is one glitch with a one for one transfer to the file. The screen is displayed in Ariel font but a text file behaves more like a Courier font. The solution is to increase the tabs by a factor of ~1.3. Then the columns will line up in the text file.

Print

It will print a report similar to a paper ICS-211 form. Use the File/Print Preview Menu command to see what it will look like.

Options

The options dialog allows specifying the Display and Printer Font Size Scale Factor, the printer orientation and the printer margins.

About

Displays the About Box. The about box uses text from the resource file (EditX.rc) for the output information. There is a standard about box dialog in the Library.

The Exit Command

Exit the application with no reservations.

Post Processing Menu

Prepare Spreadsheet

Find the first and last entry for every attendee and display the information in a manner similar to a paper ICS-211 form.

Fix Checkouts

Some attendees may not check out. This command opens a dialog box with a list of the attendees that have not checked out or the elapsed time is less than a quarter hour. The user may provide a check out time for those attendees.

Save Spreadsheet CSV File

Allows the user to chose a filename and the spreadsheet data is output in CSV format.

Tools Menu

Display Member Info

Display the contents of the current list of members.

Get Member Info

Copy the output of the RacesDB Manipulation program, ID command to the Roaming directory for the program. The ID command prepares a csv file containing the call sign, first and last name and badge number of each current member.

Edit Log Entry

Brings up a dialog box with a Combo Box containing all the entries in the ICS_211a report. Allows the use to select one where only the start and end times may be edited. Select OK when the edit has been done.

Options

Bring up the options dialog box. The options include:

  • Option for the orientation of each printed display: Portrait or Landscape
  • Option for each margin on the printed output

The margin options are a trial an error input until you find settings which are pleasing. The parameters are floating point numbers and are multiples of the width and height of an average character.

Calibrate Display/Printer

Brings up a dialog box that shows the calibration factors for the display or printer. Allows user to change them.

Help Menu

Introduction

Starts the help program with an introduction to this program.

About ICS_211a

Displays a dialog box with information about this program.
